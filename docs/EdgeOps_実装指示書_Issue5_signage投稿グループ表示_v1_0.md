## 3. 前提（C案・仕様書 v2.4 第16-1章）

- 返信投稿の `messages.body` には**返信本文だけ**が保存されている。元投稿の引用文は含まれない。
- したがって `root_post_id` を手がかりに、**表示時に元投稿を引き当てて組み立てる。**
- **元投稿が60日の取得範囲より古い場合の補完は `signage-fetch` 側で実装済み**（2026/7/22・313行）。**signage.html は返却された `messages` 配列をそのまま使えばよく、追加の取得を行わない。**

### 参照する列

| 列 | 型 | 意味 |
|---|---|---|
| `root_post_id` | uuid（NULL可） | **NULL = 元投稿。値あり = その値が指す投稿への返信** |
| `is_deleted` | boolean | 削除済みフラグ |
| `body` | text | 本文 |
| `priority` | text | `urgent` / `normal` / `info`。**値を変更しない** |
| `created_at` | timestamptz | 作成日時 |

**入れ子は構造的に発生しない**（仕様書 v1.8 第3章：返信への返信も root は元投稿に付け替えられる）。したがって**2階層だけを考えればよい。**

## 5. 実装仕様

### 5-1 投稿グループの組み立て（`loadMessage()` 内）

`signage-fetch` が返した `messages` 配列を、次の規則で畳み込む。

```
1. root_post_id が NULL の行を「元投稿」として Map に登録する（key = id）
2. root_post_id が非NULL の行を「返信」として、対応する元投稿へ紐づける
3. 紐づけ先の元投稿が配列に存在しない返信は、単独カードとして表示する（フォールバック）
4. 返信を1件も持たない元投稿は、従来どおり単独カードで表示する
5. is_deleted が true の行の扱いは、既存の単独投稿と同じ規則に従う（新しい規則を作らない）
```

**投稿グループの「最新活動日時」**＝ その元投稿と全返信の `created_at` のうち**最大値**。

### 5-2 並び替え（`messageItems.sort()`）

- 従来：`created_at` の降順
- **変更後：最新活動日時（5-1で算出した値）の降順**
- 返信を持たない投稿の最新活動日時は、その投稿自身の `created_at` とする。

**降順（新しいものが上）である点は従来と同じ。基準となる値だけが変わる。**

### 5-3 カードの表示（`buildItemHtml()`・仕様書 v2.4 第16-2章）

**サイネージ専用画面は閲覧専用であり、操作を持たない。** index.html の一覧とは異なり、**折りたたみ・開閉を設けず、簡素表示に限定する。**

| # | 規則 |
|---|---|
| 1 | 返信を持つ投稿は**1カード**にまとめ、最新活動日時の位置に置く |
| 2 | カード内は「**元投稿の要約**」「**最新返信**」「**返信件数**」の**3要素に限定**する |
| 3 | **返信ボタン・入力欄・展開操作・詳細操作を追加しない** |
| 4 | 返信が1件も無い投稿は、従来どおり単独カードで表示する |

#### 縦帯（priority）

- **元投稿の `priority`** を用いる。返信の priority は用いない。
- 既存の単独カードで使っている縦帯のCSSクラスをそのまま流用する。**新しい色・新しいクラスを追加しない。**

#### 各要素の文言

| 要素 | 内容 |
|---|---|
| 元投稿の要約 | **元投稿の本文の冒頭20文字**（20文字を超える場合は末尾に `…`） |
| 最新返信 | **最新返信の本文**（既存の単独カードの本文表示と同じ折り返し規則） |
| 返信件数 | `返信{n}件`（例：`返信3件`）。**n はその元投稿に紐づく返信の総件数** |

**元投稿が削除済みまたは取得できない場合**は、要約の位置に「**元の連絡を確認できません**」と表示する。**この場合もカード自体は表示し、最新返信は表示する。**

**注意：** signage.html は `js/i18n.js` を読み込まない。**文言はこれまでどおり signage.html 内に直接記述する。** 翻訳キーを参照する実装にしないこと。

### 5-4 モーダル表示（`openModal()` / `openModalByIndex()`）

- 投稿グループのカードを開いた場合、**モーダルにも第16-2章の3要素（元投稿の要約・最新返信・返信件数）を表示する。**
- **全返信を並べない。** 展開・折りたたみを設けない。
- `openModalByIndex()` は並び替え後の配列インデックスを参照するため、**5-2 の並び替え変更と整合していることを確認すること。**

### 5-5 index.html の関数を流用しない（第15-6章・第16-4章）

**`signage.html` は `index.html` の JS を読み込まない。** 投稿グループの組み立ても `signage.html` 内に独立して実装する。

**`buildPostGroups()` / `renderPostGroupHtml()` / `toggleRootPost()` など index.html 側の関数名をコピーして持ち込まないこと。** 名前の衝突は将来の誤読を招く。signage.html 内では別の関数名を用いる。

これは**アーキテクチャ憲法上の制約**であり、納期を理由とする見送りではない。

| # | 確認内容 | 期待値 |
|---|---|---|
| 1 | 返信の無い投稿 | 従来どおり単独カードで表示される |

| 7 | 60日より古い元投稿への返信 | 元投稿の要約が正しく出る（signage-fetch 補完取得の確認を兼ねる） |

| 9 | 引き継ぎセクション | 従来どおり表示される（無変更） |
